Fase-anker (tekst)

Fase 1
FANG

SK fanges på sketchboardet med status new. Krav: titel + desc. Fri prosa eller 6-trins-format (T-175 forslag). Plan-tilhørs valgfri på dette niveau.

Output: SK-NNN på sketchboard-data.json.

Fase 2
PRA Fase 1 — én side kendt

Per dogfood-strategy §3: én side af PRA-data identificeret. Den anden side default Medium.

  • chance_of_failure = chance_of_defects × frequency_of_use
  • damage = very low / low / moderate / high

Plan-tilhørs påkrævet: Meta / Beslutninger / Tasks / Produkter.

Output: sketch får felter plan, pra_phase: 1, og enten chance eller damage.

Fase 3
PRA Fase 2 — begge sider → Risk Class

Risk = Chance × Damage → Risk Class:

Risk ClassTest intensityQuality measures (per §4)
A (høj)IntensivMultiple coverage groups; auto-svigt-tæller; pre-action-checklist obligatorisk; sync-verify
B (moderat)Moderat1-2 coverage groups; spot-check; svigt-tæller når mønster gentages
C (lav)LetSingle coverage group; observation; ingen formel tæller

Quality characteristic identificeres per §3.5: Functionality / Maintainability / Reliability / Security / Performance / Usability.

Output: pra_phase: 2, risk_class, quality_characteristic. Status: promotion-ready.

Fase 4
DECISION GATES (§4.5)

Tjek i rækkefølge — hver gate skal være OK før næste.

GateSpørgsmålHvis fejl: Take measure
1. Data-grundlagTæller / baseline / dispatch der støtter?Etabler tæller; én-shot self-eval; bed Christian om dokumentation
2. SkillQuality i qualities.md eller knowsoft tilgængelig?Skab knowsoft via T-119 trin 1; reducer intensity; bed om hjælp eksplicit
3. EffortProportional til arbejds-omfang (T-104 / WSJF / time-budget)?Reducer intensity; pairwise i stedet for fuld; udskyd til næste cadence
4. MandatChristian godkendt scope-påvirkning?T-119 trin 2 venter; flag som forslag; afvent mandat

Hvis nogen gate fejler → take measure → tilbage til Fase 2 (re-PRA om nødvendigt).

Output: gates_status: ok eller blocked-on-{gate}.

Fase 5
PROMOTION til T-NNN trin 1

Forslag-row tilføjes til tasks-master.md som Forslag — ikke aktivt (T-119 trin 1). SK-status opdateres til promoted med reference til T-NNN.

Krav: Fase 3 + Fase 4 alle OK.

Fase 6
Christian-mandat (T-119 trin 2)

Mandat-godkendelse fra Christian. T-NNN-status skifter til Aktiv stående regel (T-119 trin 2). Test-periode start.

Fase 7
Test-periode

7-30 dage med svigt-tæller-overvågning + dogfood-anvendelse. Memory.md læseguide-tæller etableres. Bug-board overvåger via T-166.

Fase 8
Måling (§6 LEAN-loop)

Pattern-mining 28/6 + svigt-tæller-resultat. Kaizen-cyklus: fang → klassificér → tærskel (T-166: 3/7) → modgift → måling → strategi-justering.

Fase 9
KONSEKVENS — tre veje
  • Trin 3 — auto-opgave / sprite. Reglen kører selvstændigt uden Marcus-inde-over hver iteration.
  • Rollback — anomaly fanget, T-NNN rolled back. Tilbage til Fase 2 med ny PRA hvis ny vinkel.
  • Iteration — ny vinkel, ny test-periode. Fortsætter via Fase 7.
